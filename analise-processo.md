# Análise de Processo - AgileTech Solutions

## A)

A AgileTech trabalha com requisitos pouco definidos e mudanças frequentes. Por isso, os valores do Manifesto Ágil fazem muito sentido nesse cenário:

**Indivíduos e interações mais que processos e ferramentas:**  
Como a equipe é pequena, conversar diretamente resolve problemas mais rápido do que seguir processos complexos.

**Software em funcionamento mais que documentação abrangente:**  
A empresa já teve problemas com documentação desatualizada, então faz mais sentido focar em entregar algo funcionando.

**Colaboração com o cliente mais que negociação de contratos:**  
O cliente participa do processo, então é melhor manter contato frequente para validar as funcionalidades.

**Responder a mudanças mais que seguir um plano:**  
Como os requisitos mudam bastante, é importante ter flexibilidade.

### Por que Ágil e não Cascata?

O modelo Cascata exige que tudo esteja definido desde o início, o que não acontece aqui.  
Já o modelo ágil permite mudanças ao longo do desenvolvimento, o que é mais adequado para esse tipo de projeto.

### Práticas sugeridas:

- Sprints curtas (2 semanas)  
- Reuniões diárias (Daily)  
- Refinamento de backlog

## B)

Na programação em pares, duas pessoas trabalham juntas no mesmo código:
- uma escreve  
- a outra revisa  

### Benefícios

- Menos erros  
- Melhor qualidade do código  
- Troca de conhecimento  

### Desafios no remoto

- Dependência de internet    
- Cansaço em chamadas longas  

### Adaptações

- Uso de ferramentas como VS Code Live Share  
- Alternar funções com tempo controlado

## C)

As principais dificuldades nesse caso são:

Mutabilidade: requisitos mudam com frequência  
Invisibilidade: dificuldade para avaliação do progressão sem a vizualização

### Como o Ágil ajuda

- Permite mudanças durante o processo  
- Entrega partes do sistema com frequência  
- Usa ferramentas como Kanban para mostrar o andamento
